01. REST


Atualmente o mais popular, leve e flexível. Ele utiliza o protocolo HTTP para transações e trabalha nativamente com formatos de dados como JSON e XML.

02. SOAP


É um protocolo mais robusto e rígido, ideal para sistemas corporativos que exigem alta segurança e transações complexas. Ele utiliza exclusivamente o formato XML

03. XML-RPC


Uma das primeiras formas de Web Service. Utiliza XML para codificar as chamadas de procedimento remoto e HTTP para a transferência de dados.

04. GraphQL


Desenvolvido pelo Facebook, é uma alternativa moderna ao REST. Ele permite que o cliente solicite exatamente os dados específicos de que precisa, tornando a comunicação mais rápida e otimizada.
